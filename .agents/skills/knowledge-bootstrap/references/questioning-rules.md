# Questioning Rules

## 目标

在信息不足时，既避免瞎写，也避免把用户问烦。

## 追问层级

### 第一层：必须追问

缺少以下任意关键信息时，暂停正式补全：

- 公司名、品牌名、产品名、官网、文档来源全部缺失
- 不清楚本次是初始化还是增量更新
- 涉及客户案例、价格、团队背景、合作背书，但不清楚是否可公开

推荐问法：

```text
为了避免我把知识库写偏，我先确认 4 个最关键的信息：
1. 公司/品牌/产品名是什么？
2. 你希望我主要根据什么补全：文档、官网，还是公司名？
3. 这次是第一次初始化，还是在现有 knowledge 上补充更新？
4. 客户案例、价格、团队背景、合作背书里，哪些可以公开写？
```

### 第二层：强烈建议追问

这类信息不阻塞初始化，但强烈影响知识库质量：

- 品牌语气
- 明确竞品或对标账号
- 重点运营平台
- 历史内容账号主页或栏目链接
- 行业关键词侧重点
- 不能碰的话题
- 重点产品/服务优先级

优先用可选项追问。

品牌语气推荐问法：

```text
品牌语气我建议先定一个基线，你选最接近的：
1. 专业务实型（推荐）
2. 行业顾问型
3. 设计师沟通型
4. 科技先锋型

如果你不选，我先按“专业务实型”写。
```

竞品推荐问法：

```text
竞品/对标对象这块你可以直接给我：
1. 明确竞品名单
2. 你希望对标的账号或公司
3. 如果你没有，我先自动初筛 3-5 个候选，再给你确认
```

重点平台推荐问法：

```text
内容优先平台我建议先确认一下：
1. 小红书（推荐）
2. 公众号
3. 抖音
4. 三个平台都做，但要有主次顺序
```

历史内容推荐问法：

```text
如果你有小红书、公众号、抖音等历史内容，可以发我主页链接，我会补充历史数据并进行复盘；如果暂时没有，我先只生成待补充模板。
```

行业关键词推荐问法：

```text
行业关键词我建议先确认抓取重心：
1. 产品词优先（推荐）
2. 行业词优先
3. 用户痛点词优先
4. 竞品词优先

如果你不选，我先按“产品词 + 行业词混合”生成 dailyQueries。
```

### 第三层：可先推断后确认

这类内容可以先按默认策略生成，不需要阻塞流程：

- 用户画像初稿
- 常见痛点
- 候选竞品
- 可做内容方向
- 品牌语气倾向
- 候选行业关键词

这类信息不要在知识库正文里标限制性状态。如果确实需要说明信息来源或限制，写进 `source-notes.md` 的“补充记录”。

## 追问节奏

1. 首轮只问最小闭环问题，保证能开始工作。
2. 产出第一版知识库后，再追问高价值缺口。
3. 每次最多问 3-5 个问题。
4. 非阻塞问题如果用户不回复，继续产出；必要时把缺口写进 `source-notes.md` 的“补充记录”。

## 默认策略

- 没给品牌语气：默认 `专业务实型`
- 没给重点平台：默认 `小红书 > 公众号 > 抖音`
- 没给竞品：允许自动初筛候选，并在竞品文件中作为候选竞品写明判断依据
- 没给行业关键词偏好：默认 `产品词 + 行业词混合`
- 没给历史内容：只生成历史内容待补充模板和补充建议
- 给了历史内容主页但没给平台：先根据链接域名或页面标题判断平台
